# 서브 에이전트 설정 및 라우팅 전략 (Sub-Agent Strategy)

현재 구성되어 있는 서브 에이전트 기반의 능동형 오케스트레이션(Autonomous Orchestration) 및 도구 라우팅 전략입니다. 오케스트레이터(메인 에이전트)는 작업의 성격에 따라 가장 적합한 서브 에이전트와 도구를 동적으로 호출하여 작업을 수행합니다.

## 1. App/UI Development 에이전트 (`stitch` + `bkit`)

- **트리거(Trigger):** UI/UX 디자인 구성, 프론트엔드 코드(React/Next.js + Tailwind CSS) 작성이 필요할 때
- **작동 방식 (Code as Design):**
  1. `stitch` MCP 서버를 활용하여 텍스트 프롬프트를 바탕으로 시각적 화면(스크린)과 컴포넌트 변형(Variant)을 디자인하고 생성합니다.
  2. 생성된 디자인을 Next.js 기반의 프론트엔드 코드로 변환합니다.
  3. `bkit` 스타일 가이드라인과 프로젝트의 `DESIGN.md` 제약 조건(예: Glassmorphism, Grand Nature Theme)에 맞추어 표준화합니다.

## 2. Complex Tasks 에이전트 (`SequentialThinking`)

- **트리거(Trigger):** 아키텍처 설계, 다단계 추론, 복잡한 로직 구조화 및 전략 수립 시
- **작동 방식:**
  - `sequential-thinking` MCP 서버를 호출하여 문제를 세분화합니다.
  - **PDCA (계획-실행-검증-개선)** 사이클을 동적으로 적용하여, 생각의 흐름을 전개하고 가설을 검증하며 최적의 해답을 도출합니다.

## 3. Web Browser 제어 에이전트 (`browser_subagent`)

- **트리거(Trigger):** 실제 브라우저 환경에서 페이지 상호작용(클릭, 타이핑, 네비게이션 등) 및 시각적 동작 확인이 필요할 때
- **작동 방식:**
  - 주어진 목표를 달성할 때까지 스스로 웹 페이지를 조작하고 DOM 상태를 읽어들입니다.
  - 브라우저 상의 모든 상호작용은 WebP 비디오 아티팩트로 자동 기록되어 직관적인 검증이 가능합니다.

## 4. 지식 및 컨텍스트 관리 서브 에이전트 (`Knowledge` & `context7`)

- **트리거(Trigger):** 최신 프레임워크/라이브러리 공식 문서 정보가 필요하거나, 과거 프로젝트 작업 기록을 참조해야 할 때
- **작동 방식:**
  - **Context7:** 외부 라이브러리의 최신 문서와 코드 스니펫에 접근하여 지식을 보완합니다.
  - **Knowledge Items (KIs):** 시스템 백그라운드의 지식 서브 에이전트가 이전 대화 기록을 기반으로 KIs(지식 항목)를 생성 및 업데이트합니다. 메인 에이전트는 본격적인 작업 전 반드시 KI 요약본을 확인하여 중복 분석을 피하고 토큰을 절약(Context Engineering)합니다.

## 5. 데이터 및 외부 연동 에이전트

작업의 종류에 따라 아래의 특수 MCP 서버들을 연계하여 기능을 확장합니다.

- **github:** 브랜치 생성, 커밋, PR 생성 및 리뷰 등 형상 관리 통제
- **TalkToFigma:** Figma 내부의 노드/프레임 정보, 스타일, 오버라이드 값 등을 추출하여 실제 코드와 매핑
- **notebook_mcp:** (데이터/로직 전용) 계산, 데이터 분석, 비즈니스 로직 검증 시 Python 스크립트 실행으로 정밀도 확보
- **everything:** 기타 세션 리소스 관리, 로깅 시뮬레이션 및 환경 변수 통합 관리

---
**적용 원칙 (Core Rules)**

- 작업 전 `DESIGN.md` 및 생성된 Context 파일들을 우선 확인합니다.
- 단순한 설명보다 실행 결과 및 코드로 커뮤니케이션합니다.
- 모든 작업 완료 후 품질 검증 단계를 거치며, 주요 결정 사항은 문서화하여 관리합니다.
