### 3. 반응형 카드 순서 제어
**문제**:
- PC에서 화살표 흐름과 카드 배치가 불일치
- 모바일에서 순서가 꼬임

**해결**:
```javascript
// 카드 배치 순서
if (idx === 0) orderClass = 'order-1';        // 1번
else if (idx === 1) orderClass = 'order-2';   // 2번
else if (idx === 2) orderClass = 'order-3';   // 3번
else if (idx === 3) orderClass = 'order-4 lg:order-6';  // 4번 → PC에서 6번 위치
else if (idx === 4) orderClass = 'order-5';   // 5번
else if (idx === 5) orderClass = 'order-6 lg:order-4';  // 6번 → PC에서 4번 위치
```

**결과**:
- **모바일**: 1 → 2 → 3 → 4 → 5 → 6 (자연스러운 순서)
- **PC**: 1 → 2 → 3 / 6 ← 5 ← 4 (여정 흐름에 맞춤)

**파일**: `src/app/about/page.tsx` (라인 476-483)

## 🚫 제거된 요소

### 1. 복잡한 Absolute 레이어 연결선
**이유**:
- SVG path에서 퍼센트(%) 사용 불가로 오류 발생
- 위치 계산이 복잡하고 부정확
- 카드 뒤에 숨어서 보이지 않음

**대안**: 각 카드에 직접 붙은 간단한 화살표 요소

### 2. 카드 6의 루프백 곡선 화살표
**이유**:
- 왼쪽에 위치하여 시각적으로 어색함
- 복잡한 SVG 경로로 유지보수 어려움

**대안**: 하단의 "지속적인 순환 여정" 라벨로 충분히 표현

### 3. CSS Grid Order의 인라인 스타일 방식
**이유**:
- CSS 변수 방식이 복잡함
- Tailwind 클래스로 더 간단하게 처리 가능

**대안**: Tailwind의 반응형 order 클래스 사용

## 💡 주요 기술 결정

1. **SVG vs CSS 연결선**
   - 결정: CSS 기반 연결선 사용
   - 이유: SVG path 퍼센트 지원 불가, 유지보수 용이

2. **CSS Grid Order 사용**
   - 결정: Tailwind 반응형 클래스 사용
   - 이유: 간단하고 명확한 코드, 반응형 제어 용이

3. **화살표 위치**
   - 결정: 각 카드에 직접 배치
   - 이유: 정확한 위치 제어, z-index 문제 없음

4. **루프백 표현**
   - 결정: 하단 라벨로만 표현
   - 이유: 시각적 복잡도 감소, 메시지 명확성

## 🐛 해결된 버그

1. **SVG path 퍼센트 오류**
   - 문제: `d="M 25% 20%..."` 형식 지원 안 됨
   - 해결: CSS div 기반 연결선으로 대체

2. **모바일 순서 문제**
   - 문제: CSS order가 모든 화면에 적용
   - 해결: 반응형 클래스 `lg:order-*` 사용

3. **연결선 z-index 문제**
   - 문제: 카드 뒤에 숨어서 안 보임
   - 해결: 카드 내부에 직접 배치

4. **텍스트 가독성**
   - 문제: 폰트 크기가 너무 작음
   - 해결: 모든 텍스트 1단계씩 증가
