# Vercel 배포 작업 분담 및 사용자 실행 가이드

> 작성일: 2026-03-06
> 기반 문서: `vercel_deployment_next_steps.md`
> 목적: 성공적인 Vercel 배포를 위한 AI와 사용자의 역할 구분 및 사용자 상세 실행 가이드 제공

---

## 1. 역할 분담 (AI vs 사용자)

Vercel 배포 과정은 코드를 다루는 부분(AI 가능)과, 외부 서비스(Vercel, 카페24)의 대시보드에 로그인하여 설정해야 하는 부분(사용자 전담)으로 나뉩니다.

### 🤖 AI(Assistant)가 할 수 있는 작업
AI는 주로 로컬 터미널 환경과 코드 레벨에서 지원할 수 있는 작업을 담당합니다.
- **Git 커밋 및 GitHub 푸시**: 로컬에 적용된 마이그레이션 코드를 GitHub 저장소에 올립니다. (원하실 때 언제든 명령을 내려주시면 바로 실행합니다.)
- **데이터 시딩 (DB 초기화) 실행**: 배포가 완료된 후 터미널에서 스크립트를 실행하여 관리자 데이터를 서버 DB로 밀어넣습니다.
- **로컬 `.env.local` 세팅 반영**: Vercel 설정 완료 후 발급받은 키 값들을 알려주시면, 로컬 파일에 적용시켜 줍니다.
- **코드 및 API 동작 테스트**: 배포 후 터미널 명령어 수준에서 통신이 잘 되는지 점검합니다.

### 👤 사용자(개발자)가 직접 해야 하는 작업
외부 서비스의 인증 정보(비밀번호 등)와 플랫폼 UI를 조작해야 하므로 **반드시 사용자님이 직접** 하셔야 하는 작업입니다.
- **Vercel 프로젝트 생성 (GitHub 연결)**
- **Vercel Storage (Redis, Blob) 생성 및 연결**
- **비밀번호 등 환경변수 수동 세팅**
- **카페24 도메인 DNS 레코드 변경**
- **최종 모바일 및 브라우저 UI 검증**

## 2. 사용자(개발자) 상세 실행 가이드

사용자님께서 직접 진행하셔야 할 작업의 상세 가이드입니다. 아래 순서대로 천천히 따라와 주시면 됩니다. 
(이 작업을 진행하시기 전에, 저에게 **"Git 커밋 및 푸시 진행해줘"**라고 먼저 말씀해 주셔야 코드가 GitHub에 올라갑니다.)

### Phase 1: Vercel 계정 및 프로젝트 초기 세팅

1. [Vercel](https://vercel.com)에 접속하여 GitHub 계정으로 로그인합니다.
2. 메인 대시보드 우측 상단의 **[Add New...] -> [Project]** 버튼을 클릭합니다.
3. 방금 푸시된 `damha` 저장소 옆의 **[Import]** 버튼을 누릅니다.
4. **Configure Project** 화면이 나옵니다.
   - Framework Preset이 `Next.js`로 잡혀있는지 확인합니다.
   - Root Directory는 `./` 인지 확인합니다.
   - **⚠️ 주의**: 여기서 바로 `Deploy` 버튼을 누르지 마세요! 아직 데이터베이스가 연결되지 않아 빌드 에러가 납니다.

### Phase 2: 데이터베이스 및 스토리지 연결 (이후 자동 배포)

Vercel 대시보드 상단 메뉴 중 **[Storage]** 탭을 이용해 구성합니다. (최근 Vercel UI 기준으로, 프로젝트 설정화면이나 상단 탭에 Storage 메뉴가 위치합니다.)

#### 1) Redis 연결 (Upstash Redis)
1. **[Storage]** 탭으로 이동한 뒤, **[Browse Marketplace]** 창에서 **Upstash Redis**를 선택합니다.
2. 약관 동의 및 연결을 진행하고, Store 생성 옵션 창이 나오면 아래와 같이 입력/선택합니다.
   - Database Name: `damha-news` (알맞은 이름)
   - **Region: `Seoul (ap-northeast-2)`** -> 한국 타겟이므로 무조건 서울로 지정하세요!
   - Plan: `Free`
3. 생성이 완료되면 Vercel 환경변수에 `UPSTASH_REDIS_REST_URL`과 `UPSTASH_REDIS_REST_TOKEN`이 자동으로 주입됩니다.

#### 2) 이미지 저장소 연결 (Vercel Blob)
1. 다시 **[Storage]** 탭으로 돌아와 이번에는 **Vercel Blob**을 생성합니다.
2. 스토어 이름을 적절히 짓고 생성하면 됩니다.
3. 생성 완료 시 `BLOB_READ_WRITE_TOKEN` 환경변수가 자동 주입됩니다.

#### 3) 관리자 비밀번호 환경변수 수동 추가
1. 프로젝트 상단 탭에서 **[Settings]** -> 좌측 메뉴에서 **[Environment Variables]**를 클릭합니다.
2. 아래 환경변수를 수동으로 추가합니다.
   - Key: `ADMIN_PASSWORD`
   - Value: `(실제 관리자 페이지 비밀번호로 쓸 영문/숫자)`
   - Environments: `Production`, `Preview`, `Development` 모두 체크
3. **Save**를 누릅니다.

#### 4) 첫 배포 실행
1. 위 변수들이 모두 세팅되었다면, 상단 탭 **[Deployments]**로 이동합니다.
2. 처음에 실패했던 항목이 있다면 우측 점 3개(`...`)를 눌러 **[Redeploy]**를 클릭하거나 플랫폼 안내에 따라 새 배포를 시작합니다.
3. 빌드가 정상 완료되고 URL(예: `damha-xxx.vercel.app`)이 할당되면 **1차 작업 성공**입니다.

### Phase 3: 카페24 DNS 설정 (도메인 연결)

이제 우리가 가진 도메인을 Vercel에 연결해야 합니다.

1. **Vercel 세팅**:
   - Vercel 프로젝트 상단 탭 **[Settings]** -> 좌측 메뉴 **[Domains]**로 이동합니다.
   - 갖고 계신 실제 도메인 주소(예: `damha.co.kr` 또는 `www.damha.co.kr`)를 입력하고 **[Add]**를 누릅니다.
   - Vercel 화면에 **DNS 레코드를 어떻게 바꿔라(A 레코드 및 CNAME 레코드값)** 라고 표시됩니다.

2. **카페24 세팅**:
   - [카페24 호스팅센터](https://hosting.cafe24.com)에 로그인하여 **[나의 서비스 관리] -> [도메인 관리] -> [DNS 관리]** 메뉴로 들어갑니다.
   - 도메인을 선택하고 **[DNS 관리]** 버튼을 누릅니다.
   - 기존에 연결되어 있던 가비아 또는 카페24 서버 IP(A레코드) 레코드를 **삭제**합니다.
   - **레코드 추가**를 통해 Vercel이 알려준 값(아래 기본 예시 참조)을 입력합니다.
     - **타입: A**, 호스트명: (빈칸 또는 `@`), IP 주소: `76.76.21.21`
     - **타입: CNAME**, 호스트명: `www`, 목적지 도메인: `cname.vercel-dns.com`

> **참고**: DNS 변경은 전 세계 서버로 전파되는 데 통상 수 분 ~ 최대 48시간이 걸릴 수 있습니다. (요즘은 대체로 5~30분 내에 반응합니다.)

## 3. 요약 및 다음 행동 지침

모든 준비가 되셨다면 저에게 이렇게 말씀해 주시면 됩니다!

**"1단계(Git 푸시)를 먼저 진행해줘!"** 

그러면 제가 로컬의 수정한 코드들을 GitHub에 안전하게 푸시하고, 사용자님께서 위 가이드에 따라 Vercel 대시보드 작업을 진행하실 수 있도록 돕겠습니다. Vercel 세팅과 배포가 끝나면, 도메인 연결 설정과 더불어 마지막 관리자 데이터 시딩까지 제가 이어서 마무리해 드리겠습니다.
