## 2. 📝 남은 작업: 사용자 및 AI 추가 가이드

Vercel 서버 자체는 정상적으로 돌아가고 있습니다. 이제 **데이터를 채우고, 도메인을 붙이는 두 가지 작업**만 남았습니다.

### Phase 3: 카페24 DNS 설정 (도메인 연결 / 사용자 전담)

현재 임시 Vercel 도메인(`website-xxxx.vercel.app`)을 실제 소유하신 도메인(예: `damha.co.kr`)으로 연결하는 작업입니다.

#### 1) Vercel Vercel 대시보드 세팅
1. Vercel 프로젝트 상단 탭 **[Settings]** -> 좌측 메뉴 **[Domains]**로 이동합니다.
2. 가지고 계신 실제 도메인 주소를 입력하고 **[Add]**를 누릅니다.
3. Vercel 화면에 **DNS 레코드를 어떻게 바꿔야 할지(A 레코드 및 CNAME 값)** 가 안내됩니다.

#### 2) 카페24 세팅
1. [카페24 호스팅센터](https://hosting.cafe24.com)에 로그인하여 **[나의 서비스 관리] -> [도메인 관리] -> [DNS 관리]** 메뉴로 들어갑니다.
2. 도메인을 선택하고 **[DNS 관리]** 버튼을 누릅니다.
3. 기존에 연결되어 있던 가비아/카페24 등의 웹호스팅 IP(A레코드)를 **삭제**합니다.
4. **레코드 추가**를 통해 Vercel이 안내해 준 값을 추가합니다.
   - **타입: A**, 호스트명: (빈칸 또는 `@`), IP 주소: `76.76.21.21` (Vercel 기본 IP)
   - **타입: CNAME**, 호스트명: `www`, 목적지 도메인: `cname.vercel-dns.com`
5. DNS 전파를 기다립니다. (통상 5분~최대 48시간 내에 적용 완료되며, SSL 인증서가 Vercel에서 자동 발급됩니다.)

### Phase 4: 뉴스 보드 초기 데이터 시딩 (DB 복구 / AI 혹은 사용자 담당)

기능 테스트를 위해 원래 홈페이지에 박혀있던 정적 데이터 9개를 Redis 서버로 밀어넣는 작업입니다.

**✅ 진행 방법 (택 1):**

1. **AI 대행 실행 (권장)**:
   > "성공한 Vercel 도메인 주소(예: `https://website-xxxx.vercel.app`)와 관리자 비밀번호를 알려줄 테니 시딩 진행해 줘" 라고 말씀해 주시면 제가 명령어를 날려 데이터를 집어넣겠습니다.
   
2. **사용자 직접 실행**:
   만약 비밀번호 노출이 우려되신다면, 터미널(VSCode 하단 등)을 열고 아래 명령어를 직접 복사/붙여넣기하여 실행해 주시면 됩니다.
   ```bash
   curl -X POST https://[Vercel에서 발급된 혹은 실제 도메인]/api/admin/seed \
     -H "x-admin-token: [사용자가 설정한 ADMIN_PASSWORD]"
   ```
   *정상 실행 시 터미널에 `{"success": true, "count": 9}` 라는 응답이 돌아옵니다.*
