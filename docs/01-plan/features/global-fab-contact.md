# 글로벌 퀵 링크 (FabContact) 수정 계획서

## 1. 현황 및 문제점
- 문제: 페이지 우측 하단에 떠있는 퀵 링크(카카오톡 상담, 전화 상담, 상단 이동 버튼)가 메인 페이지(`/`)에만 표시되고 있습니다.
- 원인: 퀵 링크를 담당하는 `FabContact` 컴포넌트가 `src/app/page.tsx` 내부에서 렌더링되고 있어 다른 페이지(포트폴리오, 소개 페이지 등)에서는 나타나지 않습니다.

## 2. 목표
- 사용자가 사이트 내 어느 페이지를 탐색하더라도 쉽게 상담 및 상단 이동 기능을 사용할 수 있도록 퀵 링크를 **전역(Global) 컴포넌트로 변경**합니다.

## 3. 수정 방안 (수행 단계)

### 단계 1: 컴포넌트 위치 이동 (Semantic 구조화)
- 현재 `FabContact`는 홈 페이지에 종속된 `src/components/home/FabContact.tsx`에 위치해 있습니다.
- 이를 모든 페이지에서 참조하는 전역 레이아웃 컴포넌트 디렉토리인 `src/components/layout/FabContact.tsx`로 이동합니다.

### 단계 2: Global Layout (`src/app/layout.tsx`) 수정
- 전역으로 적용하기 위해 컴포넌트를 `app/layout.tsx` 에 추가합니다.
- Header와 Footer 사이의 `children` 외부(body 내부)에 배치하여 모든 라우트(페이지)에서 퀵 링크가 노출되도록 구성합니다.

### 단계 3: 기존 메인 페이지 (`src/app/page.tsx`) 정리
- `page.tsx`에 남아있는 `FabContact` 컴포넌트 렌더링 코드와 동적 import 구문을 제거하여 중복 렌더링을 방지합니다.

## 4. 기대 효과
- 사이트 전체에서 일관성 있는 사용자 경험(UX) 및 전환율(상담 신청) 증가
- 컴포넌트의 논리적 위치 변경을 통해 코드베이스의 일관성 유지 (`layout` 에 전역 요소 배치)
