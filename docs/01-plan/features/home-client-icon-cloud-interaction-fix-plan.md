# 성공의 증명 섹션 추가 수정(애니메이션 및 색상, 정렬) 계획안

## 1. 개요
* **수정 파일**: 
  - `src/components/home/PortfolioPreview.tsx` (애니메이션 옵션 및 컨테이너 정렬)
  - `src/components/ui/client-icon-cloud.tsx` (기본 색상 변경)
* **목표**:
  1. 3D 클라우드로 회전하는 고객사 텍스트 및 기본 디자인 색상을 "진한 회색"으로 변경하여 세련미 상승.
  2. 스크롤하여 해당 섹션을 벗어났다가 다시 진입할 때마다 애니메이션(텍스트 페이드-업, 클라우드 등장 등)이 1회성에 그치지 않고 **항상 재실행**되도록 개선.
  3. 좌측 텍스트 문단의 정렬을 손쉽게 컨트롤할 수 있는 가이드 작성 및 적용.

---

## 2. 세부 수정 계획

### 2.1 회전하는 고객사 요소 색상 변경 ("진한 회색")
- 3D 클라우드 내에 떠다니는 텍스트와 라인 색상이 현재는 매우 어두운 검정에 가깝게 설정되어 있습니다. 
- 이를 눈이 더 편안하고 고급스러운 **진한 회색(Dark Gray, `#4b5563` 또는 `#555555`)**으로 일괄 변경합니다.
- *참고: 마우스를 올렸을 때(Hover) 변하는 텍스트 및 아이콘 기본 색상은 기존과 같이 담하 브랜드 컬러(레드)를 유지할 계획입니다.* (수정을 원하시면 맞춤 반영 가능)

### 2.2 스크롤 진입 시 애니메이션 무한 재실행 (Re-trigger)
- **원인**: 현재 Framer Motion의 애니메이션 트리거 옵션에 `viewport={{ once: true }}`(최초 1회만 실행)가 설정되어 있습니다.
- **조치**: 해당 옵션을 `viewport={{ once: false, amount: 0.2 }}` 등으로 조절합니다.
- **결과**: 브라우저 화면에서 스크롤을 올려 해당 영역이 시야에서 완전히 사라졌다가 다시 스크롤 다운하여 나타나면, 텍스트가 순차적으로 등장하는 멋진 애니메이션 모션이 **매번 새롭게 다시 실행**됩니다.

### 2.3 좌측 영역 텍스트 문단 정렬 방법
좌측 텍스트와 버튼이 담긴 컨테이너는 Tailwind CSS 클래스를 통해 정렬 방향을 완벽하게 통제할 수 있습니다. `src/components/home/PortfolioPreview.tsx` 파일 내 좌측 컨테이너의 클래스명을 수정하여 반영합니다.

* **옵션 A (현재: 좌측 정렬)**
  문단 구성을 딱 떨어지게 만듦. `className="w-full xl:w-5/12 text-left"` 
* **옵션 B (가운데 정렬)**
  문구와 하단 버튼을 모두 중앙에 맞춤. `className="w-full xl:w-5/12 text-center flex flex-col items-center"`
* **옵션 C (우측 정렬)**
  문구와 하단 버튼을 우측에 맞춤. `className="w-full xl:w-5/12 text-right flex flex-col items-end"`

*(계획안 승인 시, 옵션(A, B, C) 중 원하시는 정렬 방식을 말씀해 주시면 해당 방식으로 즉시 코드에 반영하겠습니다.)*
