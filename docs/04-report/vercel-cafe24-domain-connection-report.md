# Vercel - Cafe24 도메인 연결 설정 보고서

## 1. 개요 및 목적
- **목표:** Vercel에 배포된 Next.js 프로젝트(`damha-web/website`)에 기존 Cafe24 도메인(`damha.co.kr`) 연결
- **핵심 제약 사항:** 웹메일 및 리셀러 서비스를 사용 중이므로 **기존 카페24 호스팅 연결을 유지**해야 함. (네임서버 통째로 변경 불가)
- **작업 일시:** 2026-03-11

---

## 2. 작업 진행 이력 및 이슈 해결

### 1단계: 초기 접근 및 문제 발견 (A 레코드 직접 수정)
1. **시도:** Vercel 권장값대로 네임서버를 변경하려 했으나, 호스팅 서비스 중단 우려로 취소.
2. **시도 2:** 카페24 DNS 관리에서 Vercel의 IP(`76.76.21.21` 및 `216.198.79.1`)로 A 레코드 직접 수정 시도.
3. **문제 발생:** 
   - 카페24 시스템 특성상 **"루트 도메인(`damha.co.kr`)의 A 레코드는 호스팅 서버 IP로 강제 고정"** 되어 있어 수동으로 변경하거나 포워딩 설정을 추가하는 것이 원천적으로 불가능함 확인.

---

### 2단계: 최적의 우회 통신망(리다이렉트) 아키텍처 수립
A 레코드 수정이 막혀있고, 호스팅을 유지해야 하는 환경에 맞춘 최적의 하이브리드 연결 방식으로 방향 선회.

- **`www.damha.co.kr` (서브도메인):** CNAME 방식을 통해 Vercel 서버로 직접 연결 (정상 라우팅)
- **`damha.co.kr` (루트 도메인):** 
  - 기존 카페24 호스팅 IP(`183.111.242.17`)로 그대로 연결되게 둠.
  - 사용자가 이 주소로 들어오면 호스팅 서버(Apache) 차원에서 `www.damha.co.kr` (Vercel)로 0.1초 만에 튕겨주도록(301 Redirect) 설정 추가.
- **결과:** 기존 호스팅/메일 무중단 운영 + 홈페이지는 Vercel 단일 엔드포인트로 서비스 (SEO 점수 분산 방지)

---

### 3단계: 최종 작업 적용 상세 내역

#### ✅ 3-1. 카페24 도메인 CNAME 갱신 (사용자 직접 진행)
- 카페24 DNS 관리 > 별칭(CNAME) 관리 메뉴에서 진행
- `www.damha.co.kr` 의 기존 목적지(`cname.vercel-dns.com`)를 Vercel 최신 프로젝트 전용 고유 주소인 **`fdf9bb8da09bc9ad.vercel-dns-017.com.`** 으로 수정 및 저장.
- *결과: DNS 쿼리 확인 결과 목적지 값이 정상적으로 전개(Propagated)되고 있음.*

#### ✅ 3-2. Vercel 도메인 대시보드 정리
- Vercel > `website` 프로젝트 > Settings > Domains
- 연결 불가능한 `damha.co.kr` 항목은 제거(Remove). 
- `www.damha.co.kr` 단일 항목만 정상적으로 남겨둠.

#### ✅ 3-3. 호스팅 301 리다이렉트 강제 설정 (AI 자동화 진행)
- 카페24 FTP 서버(`183.111.242.17`, 계정 `damha2021`) 직접 접속
- 사이트 최상단(`/www` 내)의 `.htaccess` 파일 추출 및 수정
- 아래 Vercel 포워딩 규칙을 파일 최상단에 주입 후 업로드 완료.
  ```apache
  <IfModule mod_rewrite.c>
  RewriteEngine On
  # damha.co.kr 으로 접속 시
  RewriteCond %{HTTP_HOST} ^damha\.co\.kr$ [NC]
  # 무조건 https://www.damha.co.kr/ 뒤 주소를 붙여서 이동시켜라 (301 영구 이동)
  RewriteRule ^(.*)$ https://www.damha.co.kr/$1 [L,R=301]
  </IfModule>
  ```

## 3. 남은 절차 및 현 상태 모니터링
현재 **모든 기술적 설정 및 코딩 세팅이 100% 완료된 상태**입니다.

- **현재 상태:** 카페24에서 송출하기 시작한 최신 CNAME 주소가 전 세계 인터넷 구간을 타며 퍼져나가고 있는 'DNS 전파(Propagation) 대기' 중.
- **해야 할 일:** **없음 (대기).** 
- **예상 결과:** 대기 시간이 경과하면 Vercel 대시보드 `www.damha.co.kr` 우측에 있던 붉은색 `Invalid Configuration` 뱃지가 파란색 `Valid Configuration` 체크박스로 자동으로 변하면서 HTTPS(보안 인증서) 자동 적용 및 실 접속이 원활하게 진행됩니다.

## 4. 추후 관리 주의사항
- 카페24에서 `damha.co.kr`에 대한 **네임서버**를 절대 훼손(변경)해서는 안 됩니다. 변경 시 메일 및 타 호스팅 서비스에 장애가 발생합니다.
- 카페24 웹 FTP를 이용해 루트 경로(`.htaccess` 파일) 작업 시, 최상단의 Vercel 포워딩 규칙이 지워지지 않도록 유의해주세요.

## 5. 추가 이슈 및 Plan B (진행 중)
### 🚨 와일드카드 CNAME 충돌 이슈 발견
- Vercel에서의 빨간불(Invalid Configuration)이 장시간 해소되지 않는 원인을 파악한 결과, 기존 카페24 DNS 설정에 **와일드카드 CNAME(`*.damha.co.kr`)** 레코드가 우선 적용되어 있어 Vercel을 향한 개별 A 레코드/CNAME 설정값 반영을 강제로 덮어쓰고(방해하고) 있는 것으로 추측됩니다.
- 또한 리셀러 서비스 연동으로 인해 해당 와일드카드 레코드를 사용자가 임의로 삭제하거나 수정할 수 없게 막혀 있는 한계점이 있습니다.

### 🛠️ 현재 대응 및 향후 계획 (Plan B)
- **현재 대응:** 카페24 고객센터에 해당 와일드카드 CNAME 설정이 존재하는 상황에서 특정 서브도메인(`www.damha.co.kr`)만 외부 IP(Vercel)로 예외 연결이 가능한지 1:1 문의를 접수한 상태입니다.
- **Plan B (신규 도메인 구매):** 만약 리셀러 서비스 구조상 카페24 측에서 이 예외 처리를 지원하지 못한다고 답변이 올 경우, 더 이상 기존 도메인(`damha.co.kr`) 설정에 시간을 지체하지 않고 **Vercel 전용 신규 도메인을 하나 구매하여 깔끔하게 다이렉트로 연결**하는 방식으로 우회하기로 결정하였습니다.
- 신규 도메인 연결 시에는 호스팅 제약이 전혀 없으므로 네임서버 방식(방식 1)을 사용하여 가장 쉽고 강력한 보안 구성으로 5분 내 설정이 완료될 수 있습니다.
